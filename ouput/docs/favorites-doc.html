역할즐겨찾기 메뉴 관리 및 빠른 재주문

💡 설계 의도

🎯 이 화면의 핵심 목적

반복 주문 고객이 매번 메뉴를 탐색하지 않고 저장해둔 메뉴를 1-2탭으로 즉시 재주문할 수 있는 숏컷 화면입니다. 자주 시키는 메뉴를 미리 등록해두면 옵션 선택 과정 없이 바로 주문 흐름에 진입할 수 있습니다.

UX 설계 근거

  • 체크박스 선택 방식 — 여러 메뉴를 한 번에 선택하여 동시 주문이 가능하도록 설계했습니다. 개별 메뉴마다 주문하는 번거로움을 제거합니다
  • 저장된 옵션 표시 — 아이스/라지/일회용컵 등 이전에 저장한 커스텀 옵션이 리스트에 함께 표시되어 별도 확인 없이 바로 주문할 수 있습니다
  • 2가지 액션 버튼 — "장바구니 담기"와 "지금 바로 주문"으로 유연한 주문 동선을 제공합니다. 다른 메뉴를 추가하고 싶으면 장바구니에, 즉시 결제하려면 바로 주문을 선택합니다
  • 편집 기능 분리 — 즐겨찾기 편집(삭제/순서변경)은 마이페이지에서 처리하도록 분리하여 이 화면에서는 주문 흐름에만 집중할 수 있습니다

사용자 시나리오

빠른 재주문즐겨찾기 진입 → 메뉴 체크 → 지금 바로 주문 → 주문 정보 입력
장바구니에 추가즐겨찾기 진입 → 메뉴 체크 → 장바구니 담기 → 다른 메뉴 추가 탐색

🧩 화면 구성요소

상단 헤더 (Header)

뒤로가기(메뉴홈) 버튼, 홈 버튼, 중앙 "즐겨찾는 메뉴" 타이틀로 구성됩니다. 뒤로가기와 홈 모두 메뉴 홈 화면으로 이동합니다.

  • 좌측 뒤로가기 아이콘 → 메뉴 홈으로 복귀
  • 홈 아이콘 → 메뉴 홈으로 이동

총 개수 표시

현재 등록된 즐겨찾기 메뉴의 총 개수를 "총 N개" 형태로 표시합니다. 사용자가 저장된 메뉴 수를 한눈에 파악할 수 있습니다.

편집 안내 문구

"즐겨찾는 메뉴 편집은 마이페이지에서 가능합니다" 텍스트와 함께 "편집하기" 링크를 제공합니다. 주문 흐름과 편집 흐름을 분리하여 사용자가 혼동 없이 원하는 작업에 집중할 수 있도록 합니다.

즐겨찾기 리스트

저장된 메뉴를 카드 형태의 리스트로 표시합니다. 각 항목은 다음 요소로 구성됩니다.

  • 체크박스 — SVG 커스텀 버튼으로 선택/해제 상태를 토글합니다
  • 메뉴 썸네일 — 메뉴 이미지를 소형으로 표시합니다
  • 메뉴명 — 저장된 메뉴의 이름을 표시합니다
  • 가격 — 저장 시점의 옵션이 반영된 가격을 표시합니다
  • 저장된 옵션 — "아이스/라지/일회용컵" 등 커스텀 옵션을 텍스트로 표시합니다

하단 액션 버튼

화면 하단에 고정되는 2개의 액션 버튼입니다. 선택된 메뉴에 대해 즉시 실행됩니다.

장바구니 담기 지금 바로 주문
  • 장바구니 담기 — 선택한 메뉴를 장바구니에 추가하고, 계속 탐색할 수 있습니다
  • 지금 바로 주문 — 선택한 메뉴로 즉시 주문 정보 입력 화면으로 이동합니다
