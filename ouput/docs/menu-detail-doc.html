역할개별 메뉴의 상세 정보 제공 및 주문 옵션 선택

💡 설계 의도

🎯 이 화면의 핵심 목적

사용자가 메뉴를 선택하기 전 충분한 정보를 확인하고, 옵션(온도/사이즈/원두/컵)을 선택한 후 주문까지 한 화면에서 완료할 수 있도록 설계했습니다.

UX 설계 근거

  • 대형 상품 이미지 — 히어로 이미지로 시각적 기대감을 형성하여 구매 전환율을 높입니다
  • 순차적 정보 노출 — 가격/설명/주의사항을 단계적으로 배치하여 정보 과부하를 방지합니다
  • 영양 정보 탭 전환 — 온도/사이즈별 영양성분을 탭으로 분리하여 건강 관심 고객에 대응합니다
  • 알레르기 유발요인 별도 섹션 — 안전 정보를 명확하게 분리 제공하여 사용자 신뢰를 확보합니다
  • 바텀시트 주문 플로우 — 옵션 선택 → 수량 조절 → 주문하기 CTA의 자연스러운 흐름으로 이탈을 최소화합니다

사용자 시나리오

정보 확인 후 주문상품정보 확인 → 주문하기 → 옵션선택 → 주문하기
장바구니 담기상품정보 확인 → 주문하기 → 옵션선택 → 장바구니 아이콘
영양정보 확인스크롤다운 → 온도/사이즈 탭 전환 → 영양성분 확인

🧩 화면 구성요소

1. 상단 헤더 (Header)

좌측에 뒤로가기 버튼(카테고리)과 홈 버튼, 중앙에 "커피/음료" 헤더 타이틀, 우측에 장바구니 아이콘이 배치됩니다.

  • 뒤로가기 → 카테고리 화면 (category-coffee.html)
  • 홈 → 메뉴 홈 (menu.html)
  • 장바구니 아이콘에 담긴 수량 뱃지 표시

2. 상품 히어로 이미지 (Hero Image)

전체 너비 상품 사진이 중앙 정렬로 표시됩니다. 대형 이미지를 통해 메뉴에 대한 시각적 기대감을 형성하고 구매 욕구를 자극합니다.

3. 상품 정보 섹션 (Product Info)

상품의 핵심 정보를 순차적으로 노출하는 영역입니다.

  • New 뱃지 — 신메뉴 여부에 따라 조건부 표시 (isNew)
  • 상품명 — 메뉴 이름 (22px / Bold)
  • 설명 — 메뉴 상세 설명 텍스트
  • 가격 — 숫자 포맷 적용 (20px / Bold)
  • 카페인 주의사항 — 카페인 함유 메뉴에 대한 안내 문구

4. 영양 정보 (Nutrition Info)

온도와 사이즈에 따라 영양 정보를 탭으로 전환하여 확인할 수 있는 인터랙티브 섹션입니다.

  • 온도 탭 — 핫 / 아이스 전환
  • 사이즈 탭 — 레귤러 / 라지 전환
  • 영양 테이블 — 1회 제공량, 열량, 당류, 단백질, 포화지방, 나트륨, 카페인
  • 면책 조항 텍스트 하단 표시

5. 알레르기 유발요인 섹션 (Allergens)

해당 메뉴의 allergens 데이터를 기반으로 알레르기 유발요인 정보를 별도 섹션으로 제공합니다. 식품 안전 정보를 명확히 분리하여 사용자가 쉽게 확인할 수 있습니다.

6. 하단 CTA "주문하기" 버튼

화면 하단에 고정된 주문 진행 버튼입니다. 클릭 시 옵션 선택 바텀시트가 열리며, 전체 주문 플로우의 시작점 역할을 합니다.

7. 옵션 선택 바텀시트 (Bottom Sheet)

"주문하기" CTA 클릭 시 나타나는 오버레이 바텀시트입니다. 주문에 필요한 모든 옵션을 한곳에서 선택할 수 있습니다.

  • 딤 오버레이 — 탭하면 바텀시트 닫기
  • 온도 선택 — 핫/아이스 옵션 그룹
  • 사이즈 선택 — 레귤러/라지 옵션 그룹
  • 원두 선택 — 블랙그라운드/아로마노트/디카페인 옵션 그룹
  • 컵 선택 — 매장컵/개인컵/일회용컵 옵션 그룹
  • 퍼스널 옵션 링크 — 추가 커스터마이징 접근
  • 수량 조절 — +/- 버튼으로 수량 컨트롤 (최소 1)
  • 액션 버튼 — 장바구니 담기 / 즐겨찾기 / 주문하기

🔗 인터랙션 & 화면 전환

진입 경로

  • 카테고리 메뉴 리스트에서 아이템 탭
  • 메뉴홈 최근주문 / Quick Order 탭

주문 플로우

상품정보 확인 → 주문하기 탭 → 바텀시트 오픈 → 옵션 선택 → 수량 설정 → 주문/장바구니

이 화면에서 이동 가능한 곳

주문하기 버튼→ 바텀시트 오픈 (옵션 선택)
바텀시트 주문하기→ 주문정보 입력 (order.html?id=&qty=&options=)
장바구니 담기→ localStorage에 저장 (바텀시트 내 아이콘)
즐겨찾기→ 즐겨찾기에 추가 (바텀시트 내 아이콘)
뒤로가기→ 카테고리 (category-coffee.html)
홈→ 메뉴 홈 (menu.html)
장바구니 헤더→ 장바구니 (cart.html)

⚙️ 데이터 & 기술 스펙

URL 파라미터 ?id={menuId}로 메뉴를 식별하며, MENU_DATA에서 id로 아이템을 조회합니다.

데이터 소스menu-data.js → MENU_DATA 배열에서 id 매칭
URL 파라미터?id={menuId}로 메뉴 식별
옵션 활성화updateSubmitButton()으로 CTA 활성화 + 가격 표시
수량 제어orderQty 변수, +/- 1 버튼 (최소 1)
바텀시트transform + opacity 애니메이션, 딤 오버레이
최대 화면 너비430px (모바일 최적화)

주요 데이터 필드

id고유 식별자
name / description메뉴명, 메뉴 설명
price가격 (원)
image상품 이미지 파일명
isNew신메뉴 여부 (New 뱃지 조건)
allergens알레르기 유발요인 목록
영양 정보servingSize, calories, sugar, protein, satFat, sodium, caffeine
옵션tempOptions, sizeOptions, beanOptions, cupOptions
