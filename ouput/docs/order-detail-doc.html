💡 설계 의도

🎯 이 화면의 핵심 목적

결제 완료 후 사용자에게 "주문이 정상적으로 접수되었다"는 안심감을 주고, 실시간 준비 상태를 추적할 수 있는 최종 피드백 화면입니다.

UX 설계 근거

  • 주문번호 + 주문시각 배너 — 최상단에 배너 형태로 강조 배치하여 "주문이 접수되었다"는 사실을 즉각적으로 확인할 수 있게 합니다
  • 4단계 진행 바 — 주문완료 → 준비중 → 준비완료 → 수령완료의 시각적 스텝퍼로 현재 상태를 한눈에 파악할 수 있습니다
  • 매장 정보 + 전화 아이콘 — 수령 장소를 재확인하고, 전화 아이콘으로 매장에 바로 연락할 수 있도록 합니다
  • 주문 상품 아코디언 — 1.5초 후 자동 접힘으로 상세 확인이 가능하면서도 화면 간결성을 유지합니다
  • 프로모션 배너 슬라이더 — 대기 시간 중 추가 마케팅 노출을 위한 5장 배너 자동 전환 영역입니다
  • "영수증 보기" 버튼 — 결제 내역에 대한 접근 경로를 하단에 배치하여 필요 시 확인할 수 있습니다

사용자 시나리오

주문 상태 확인결제 후 → 진행 상태 스텝퍼 확인 → 준비완료 대기
주문 내역 확인아코디언 펼치기 → 주문한 메뉴/수량/가격 확인
영수증 확인하단 "영수증 보기" 탭 → 결제 내역 접근

🧩 화면 구성요소

상단 헤더 (Header)

좌측 뒤로가기(메뉴홈으로 이동) 및 홈 버튼, 중앙에 "주문상세" 타이틀이 배치된 고정 헤더입니다.

  • 뒤로가기 버튼 → 메뉴 홈(menu.html)으로 이동
  • 홈 버튼 → 메뉴 홈(menu.html)으로 이동
  • 헤더 높이: 52px, 하단 1px 보더로 콘텐츠 영역과 구분

주문번호 배너 (Order Number Banner)

주문번호와 주문일시를 최상단에 강조 표시하는 배너 영역입니다. 결제가 완료되었다는 즉각적인 피드백을 제공합니다.

  • 주문번호: 랜덤 생성된 고유 번호 표시
  • 주문일시: getNow() 함수로 현재 시각 표시

주문 진행 상태 (Order Status Stepper)

4단계 스텝퍼로 현재 주문 상태를 시각화합니다. 각 단계마다 아이콘 상태가 다르게 표시됩니다.

주문완료 ✓ → 준비중 ● → 준비완료 ○ → 수령완료 ○
  • 완료 단계: 체크 아이콘(✓) + 활성 색상
  • 현재 단계: 채워진 원(●) + 진행 중 표시
  • 대기 단계: 빈 원(○) + 비활성 색상
  • 새로고침 버튼 → 주문 상태 갱신 시뮬레이션

프로모션 배너 슬라이더 (Promotion Banner Slider)

5장의 프로모션 배너가 자동 전환되는 슬라이더 영역입니다. 대기 시간 중 추가 마케팅 노출 목적으로 배치되었습니다.

  • 5장 배너 + 페이지네이션(1/5) 표시
  • 3.5초 간격 자동 전환
  • 터치 스와이프 제스처 지원

매장 정보 (Store Info)

현재 주문한 매장의 이름과 전화 아이콘이 표시됩니다. 수령 장소를 재확인하고 매장에 전화 연결할 수 있습니다.

주문 상품 아코디언 (Order Items Accordion)

주문한 상품의 상세 정보를 펼치기/접기 가능한 아코디언 형태로 표시합니다. 진입 시 1.5초 후 자동으로 접힙니다.

  • 토글 버튼으로 펼치기/접기 제어
  • 메뉴명(온도/사이즈 약어) 표시
  • 선택된 옵션 정보 표시
  • 가격, 수량, 소계 표시
  • 자동 접힘: setTimeout 1.5초

하단 "영수증 보기" 버튼 (Receipt Button)

화면 하단에 고정 배치된 "영수증 보기" 버튼입니다. 결제 내역 확인을 위한 접근 경로를 제공합니다.
