역할주문 아이템 관리 및 디저트 크로스셀링

🛒 장바구니

🎯 핵심 목적

여러 메뉴를 모아두고 수량/옵션을 최종 조정한 뒤 한 번에 주문할 수 있는 중간 허브 화면입니다. 충동구매를 유도하는 디저트 추천 바텀시트로 객단가 상승 기회를 제공합니다.

⚠️ 이전의 문제점

  • 수량 변경 시 별도 화면 진입 — 기존 투썸오더 앱에서는 장바구니 아이템의 수량을 변경하려면 메뉴 상세 화면으로 재진입해야 하여, 단순 수량 조정에도 불필요한 화면 전환이 발생했음
  • 크로스셀링 기회 부재 — 장바구니에서 바로 결제로 넘어가는 단순 흐름으로, 디저트 등 보완 메뉴 추가 구매를 유도할 수 있는 접점이 없어 객단가 상승 기회를 놓치고 있었음
  • 주문 유형 혼재 — 일반 주문(투썸오더)과 케이크 예약이 하나의 리스트에 혼재되어, 사용자가 주문 유형별로 관리하기 어렵고 실수로 잘못된 유형으로 결제할 가능성이 있었음

💡 UX 설계 근거

  • 탭 구조(투썸오더/케이크예약) — 주문 유형을 명확히 구분하여 일반 주문과 케이크 예약 장바구니를 분리 관리합니다
  • 매장 정보 접기/펼치기 — 매장명과 주소를 토글 방식으로 표시해 공간 효율을 극대화합니다
  • 전체선택/선택삭제 — 대량의 장바구니 아이템을 한 번에 관리할 수 있는 편의성을 제공합니다
  • 개별 아이템 컨트롤 — 수량조절(+/-), 개별삭제, 체크박스를 아이템마다 배치해 세밀한 관리가 가능합니다
  • 바코드 말풍선 — QR 버튼 옆 말풍선으로 매장 결제 안내를 직관적으로 전달합니다
  • 디저트 추천 바텀시트 — 주문하기 클릭 시 크로스셀링 기회를 제공하여 객단가를 높입니다

📋 사용자 시나리오

단일 메뉴 주문장바구니 → 수량 확인 → 주문하기 → 디저트 추가/스킵 → 결제
여러 메뉴 관리장바구니 → 불필요 메뉴 삭제 → 수량 조정 → 주문하기
메뉴 추가장바구니 → "메뉴 추가하기" → 메뉴 탐색 → 장바구니 복귀

✅ 기대 성과
  • 장바구니 관리 단계 40% 단축 — 수량/옵션 변경을 위한 별도 화면 진입 없이 장바구니 내에서 즉시 조정 가능하여 주문 완료까지의 단계를 5단계에서 3단계로 단축
  • 객단가 상승 기회 확보 — 디저트 추천 바텀시트를 통한 크로스셀링으로 추가 메뉴 구매를 유도하여 건당 주문 금액 증가 기대
  • 주문 오류율 감소 — 주문 유형별 탭 분리와 전체선택/선택삭제 기능으로 장바구니 관리 정확도를 높여 잘못된 주문 발생률 감소

※ 산정 기준: 화면 전환(탭) 수 기준 — 기존 = 장바구니 → 메뉴 상세 재진입 → 수량 변경 → 장바구니 복귀 → 결제(5단계), 개선 = 장바구니 내 즉시 조정 → 디저트 추천 → 결제(3단계)

🧩 화면 구성요소

상단 헤더 (Header)

좌측 뒤로가기, 중앙 "장바구니" 타이틀, 우측 홈 버튼으로 구성된 고정 헤더입니다. 명확한 화면 식별과 빠른 네비게이션을 제공합니다.

  • 뒤로가기 버튼으로 이전 화면 복귀
  • 홈 버튼으로 메뉴 메인 화면 이동

탭 메뉴 (Order Type Tabs)

투썸오더(일반 주문)와 케이크예약 두 개의 탭으로 장바구니 유형을 구분합니다. 활성 탭 하단에 인디케이터 바가 표시됩니다.

투썸오더 케이크예약

매장 정보 (Store Info)

현재 선택된 매장명과 주소를 표시합니다. 접기/펼치기 토글로 공간을 효율적으로 활용하며, 주문 전 매장을 한 번 더 확인할 수 있습니다.

전체선택 / 선택삭제 바

좌측에 전체선택 체크박스, 우측에 선택삭제 버튼을 배치합니다. 다수의 아이템을 한 번에 관리할 때 사용하는 일괄 제어 영역입니다.

  • 전체선택 토글 시 모든 아이템의 체크박스가 동기화
  • 선택삭제 버튼으로 체크된 아이템만 일괄 제거

장바구니 아이템 리스트 (Cart Items)

각 아이템은 체크박스, 메뉴명, 옵션 정보, 삭제 버튼, 썸네일 이미지, 가격 내역, 수량 조절 버튼(+/-)으로 구성됩니다. 개별 아이템 단위로 세밀한 관리가 가능합니다.

  • 체크박스로 주문 대상 아이템 선택/해제
  • 수량 +/- 버튼으로 실시간 수량 조정 및 금액 재계산
  • X 버튼으로 개별 아이템 삭제
  • 메뉴 썸네일 이미지와 옵션 텍스트로 메뉴 식별

메뉴 추가하기 링크

아이템 리스트 하단에 위치한 텍스트 링크입니다. 탭 시 메뉴 홈(menu.html)으로 이동하여 추가 메뉴를 탐색할 수 있습니다.

픽업매장 안내 바

선택된 픽업 매장 정보를 다시 한번 보여주는 안내 영역입니다. 결제 직전 매장 확인 기능을 합니다.

하단 주문 바 (Bottom Order Bar)

화면 하단에 고정되는 주문 액션 영역입니다. 좌측에 QR 버튼과 바코드 말풍선, 우측에 주문하기 버튼(총 금액 + 수량 표시)이 배치됩니다.

  • QR 버튼 — 매장 바코드 결제 안내
  • 바코드 말풍선 — 매장 결제 방법을 직관적으로 안내
  • 주문하기 버튼 — 선택된 아이템의 총 금액과 수량을 표시, 탭 시 디저트 추천 바텀시트 오픈

디저트 추천 바텀시트 (Dessert Upsell Sheet)

주문하기 버튼 클릭 시 올라오는 바텀시트입니다. "함께 주문하면 더 좋아요!" 타이틀과 함께 디저트 그리드, 함께 주문하기 버튼으로 구성됩니다.

  • overlay + sheet 구조로 slide-up 애니메이션 적용
  • 디저트 아이템 그리드로 크로스셀링 기회 제공
  • 함께 주문하기 버튼 → 주문 정보 입력 화면으로 이동
  • overlay 탭 또는 닫기로 바텀시트 해제

⚙️ 데이터 & 기술 스펙

장바구니 데이터는 localStorage의 twosome_cart 키를 기반으로 관리됩니다. 메뉴 정보는 MENU_DATA와 DESSERT_DATA에서 조회합니다.

데이터 저장소localStorage → twosome_cart 키
메뉴 데이터menu-data.js → MENU_DATA + DESSERT_DATA 배열
데모 샘플메뉴ID 9, 8 (2개 아이템 기본 세팅)
이벤트 처리이벤트 위임 패턴 (data-action 속성 기반)
바텀시트overlay + sheet 구조, slide-up 애니메이션
최대 화면 너비430px (모바일 최적화)

사용 기술

HTML5 CSS3 Vanilla JS localStorage 이벤트 위임 바텀시트 UI
