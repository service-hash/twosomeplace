🎯 이 화면의 핵심 목적

주문 직전 모든 결제 관련 정보를 한 화면에서 입력 및 확인하고 결제를 완료하는 최종 게이트웨이입니다. 이탈을 최소화하면서도 필요한 모든 정보를 수집하는 밸런스가 핵심입니다.

UX 설계 근거

  • 주문정보 섹션(접기/펼치기) — 선택한 메뉴를 재확인할 수 있도록 토글 구조를 제공하여, 정보 과부하 없이 필요 시 상세 내역을 확인할 수 있습니다
  • 프로모션/쿠폰 사용 진입점 노출 — 결제 전 할인 기회를 인지시켜 사용자 만족도를 높이고, 쿠폰 미사용으로 인한 결제 후 불만을 방지합니다
  • 포장 여부 라디오 버튼 — 간단한 2택(전체 포장/포장 안 함) 선택으로 의사결정 부담을 최소화합니다
  • T우주패스/기프트카드 등 다양한 결제수단 지원 — 멤버십과 선불카드를 먼저 차감하여 실결제 금액을 줄이는 전략적 배치입니다
  • 결제수단 그리드(3x2) — 투썸페이/신용카드/간편결제 등 6종을 시각적으로 한눈에 비교 선택할 수 있습니다
  • 최종 결제금액 강조 표시 — 할인 적용 후 실결제 금액을 굵은 폰트로 강조하여 결제 전 최종 확인을 유도합니다
  • 유의사항 동의 — 접기/펼치기 구조로 법적 요건을 충족하면서도 화면 공간을 효율적으로 사용합니다

사용자 시나리오

표준 결제주문정보 확인 → 결제수단 선택(신용카드) → 결제하기
쿠폰 적용프로모션 쿠폰 사용 → 할인 적용 → 결제
멤버십 활용T우주패스 적용 → 기프트카드 잔액 사용 → 잔액 카드결제

🧩 화면 구성요소

1. 상단 헤더 (Header)

좌측에 뒤로가기(동적 href) 및 홈 버튼, 중앙에 "주문정보입력" 타이틀, 우측에 장바구니 아이콘이 배치됩니다. 뒤로가기 링크는 진입 경로(from 파라미터)에 따라 동적으로 설정됩니다.

  • from=cart이면 장바구니(cart.html)로 복귀
  • 그 외에는 해당 메뉴 상세(menu-detail.html)로 복귀
  • 헤더 높이: 52px, 하단 보더로 콘텐츠 영역과 구분

2. 주문정보 섹션

접기/펼치기 토글을 지원하며, 주문 아이템 카드가 표시됩니다. 각 아이템 카드에는 메뉴명, 옵션 태그, 썸네일(60x60px), 상품금액, 퍼스널 옵션, 수량 및 주문금액이 포함됩니다.

  • 메뉴 썸네일 이미지 (60x60px, border-radius: 10px)
  • 메뉴명 + 옵션 태그 (핫/아이스, 레귤러/라지 등)
  • 상품금액 및 퍼스널 옵션 표시
  • 수량 x 주문금액 합계
  • 쿠폰/프로모션 "사용하기" 버튼

3. 페어링/영수증 쿠폰 섹션

프로모션 혜택을 적용할 수 있는 "사용하기 >" 링크가 표시됩니다. 결제 전 할인 기회를 놓치지 않도록 시각적으로 노출합니다.

4. 포장 여부

라디오 버튼 방식으로 "전체 포장" 또는 "포장 안 함" 중 택1합니다. 선택된 항목에는 체크 아이콘이 표시되어 현재 상태를 명확히 인지할 수 있습니다.

전체 포장 포장 안 함

5. T 우주패스 / T 멤버십

멤버십 카드번호를 입력하거나 확인하는 섹션입니다. T우주패스 연동 시 추가 할인 혜택이 적용됩니다.

6. 기타 결제수단

모바일 쿠폰과 기프트카드 잔액을 확인하고 사용할 수 있는 영역입니다.

  • 모바일 쿠폰 — 사용가능 수량 및 금액 표시
  • 기프트카드 — 보유 잔액 표시 + "사용" 버튼

7. 결제수단 선택

6개의 결제 수단 버튼이 3x2 그리드로 배치됩니다. 선택된 결제 수단에 활성 스타일(빨간 테두리)이 적용되며, 카드 선택 드롭다운과 "다음에도 동일한 결제수단" 체크박스가 제공됩니다.

투썸페이 신용카드 KB Pay 카카오페이 네이버페이 페이코
  • 카드 선택 드롭다운 (높이 48px)
  • "다음에도 동일한 결제수단" 체크박스
  • 매장별 간편결제 안내 문구
  • 제휴 할인 안내 문구

8. 최종 결제금액 섹션

쿠폰/할인 적용 후 최종 결제 금액이 강조 표시됩니다. 20px Bold 폰트로 금액을 부각하여 결제 전 최종 확인을 유도합니다.

9. 유의사항

접기/펼치기 토글을 지원하며, 주문 및 결제 관련 동의 사항이 포함됩니다. 법적 요건 충족을 위한 필수 영역입니다.

10. 하단 CTA "X원 결제하기" 버튼

"N원 결제하기" 형식의 CTA 버튼이 화면 하단에 배치됩니다. 결제 금액이 동적으로 반영되며, 버튼 클릭 시 주문 완료 화면(order-detail.html)으로 이동합니다.

  • CTA 높이: 54px, border-radius: 12px
  • Primary Red 배경(#d42a2a), 흰색 텍스트
  • 17px Bold 폰트

🔗 인터랙션 & 화면 전환

진입 경로

  • 단품 주문 — 메뉴 상세에서 "주문하기" 버튼 클릭 (?id=&qty=&options=)
  • 복수 주문 — 장바구니 바텀시트에서 "주문하기" 클릭 (?from=cart)

결제 흐름

주문 정보 확인 → 쿠폰/포장 선택 → 결제 수단 선택 → 결제하기 → 주문 완료

이 화면에서 이동 가능한 곳

뒤로가기 (from=cart)→ 장바구니 (cart.html)
뒤로가기 (단품)→ 메뉴 상세 (menu-detail.html)
홈 버튼→ 메뉴 메인 (menu.html)
장바구니 아이콘→ 장바구니 (cart.html)
결제하기 CTA→ 주문 완료 (order-detail.html)

⚙️ 데이터 & 기술 스펙

URL 파라미터

단품 주문?id=&qty=&options= — 메뉴 ID, 수량, 옵션(URL 인코딩)
장바구니 주문?from=cart — localStorage에서 주문 데이터 읽기

데이터 소스

상품 조회MENU_DATA + DESSERT_DATA에서 아이템 정보 조회
장바구니 데이터localStorage twosome_order 키에서 장바구니 주문 데이터 읽기
결제 저장결제 시 localStorage twosome_order_detail에 주문 데이터 저장

동적 뒤로가기

URL의 from 파라미터에 따라 뒤로가기 링크의 href를 동적으로 설정합니다. from=cart이면 cart.html, 그 외에는 menu-detail.html?id=N으로 분기합니다.
