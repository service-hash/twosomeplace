✨ NEW 탭 — 신메뉴 탐색

🎯 핵심 목적

새롭게 출시된 메뉴를 한 곳에 모아 보여주어, 신메뉴 발견과 시도를 촉진하는 큐레이션 전용 탭입니다.

⚠️ 이전의 문제점

  • 신메뉴 발견성 부재 — 기존 앱에서는 신메뉴가 전체 카테고리 목록에 섞여 노출되어, 사용자가 새로운 메뉴를 별도로 확인할 수 있는 전용 경로가 없었음
  • 시즌 메뉴 탐색 비효율 — 시즌 한정 메뉴나 프로모션 메뉴를 발견하려면 전체 카테고리를 순회하며 직접 찾아야 했으며, 놓치는 경우가 빈번했음
  • 건강 니즈 필터 부재 — 저당·저칼로리·카페인 프리 등 건강 관련 조건으로 신메뉴를 필터링할 수 있는 기능이 없어, 개인화된 탐색이 불가능했음

💡 UX 설계 근거

  • NEW 전용 탭 분리 — 메뉴 탐색 영역 상단에 독립적인 NEW 탭을 배치하여, 한 번의 탭으로 최신 메뉴를 모아볼 수 있는 전용 진입점 제공
  • NEW 뱃지 시각 강조 — 각 신메뉴 카드에 빨간색 NEW 뱃지를 부착하여, 시각적으로 즉시 신메뉴임을 인지할 수 있도록 설계
  • 필터 칩 시스템 — 저당·저칼로리·카페인 프리 3종의 필터 칩을 탭 상단에 배치하여, 건강 관련 조건 기반 탐색 지원 — 복수 선택 가능하여 개인 니즈에 맞는 세밀한 필터링 가능

📋 사용자 시나리오

신메뉴 탐색Order 탭 → NEW 탭 전환 → 최신 출시 메뉴 한눈에 확인 → 관심 메뉴 선택
건강 필터 탐색NEW 탭 → '저칼로리' 필터 칩 활성화 → 저칼로리 신메뉴만 필터링하여 확인
시즌 메뉴 발견NEW 탭 진입 → 시즌 한정 메뉴 발견 → 상세 정보 확인 → 장바구니 추가

📈 기대 효과

✅ 기대 성과
  • 신메뉴 인지율 향상 — 전용 탭 진입점을 통해 신메뉴 노출 빈도를 높이고, NEW 뱃지로 시각적 주목도를 강화하여 인지율 대폭 향상
  • 시즌 메뉴 시도율 증가 — 한정 메뉴를 놓치지 않고 발견할 수 있는 전용 경로 제공으로 시즌 메뉴 주문 전환율 향상 기대
  • 건강 기반 탐색 만족도 개선 — 필터 칩을 통한 개인화 탐색으로 건강 관심 사용자의 탐색 효율성 및 만족도 향상

☕ 커피/음료 탭 — 카테고리별 메뉴 탐색

🎯 핵심 목적

전체 음료 메뉴를 6개 서브 카테고리로 체계적으로 분류하여, 원하는 메뉴를 빠르게 찾을 수 있도록 설계한 주 카테고리 탭입니다.

⚠️ 이전의 문제점

  • 평면적 메뉴 나열 — 기존 앱에서는 모든 음료 메뉴가 단일 리스트로 나열되어, 수십 개의 메뉴 중 원하는 종류를 찾기 위해 긴 스크롤이 필요했음
  • 카테고리 세분화 부족 — 커피·디카페인·음료·티·아이스크림 등 특성이 다른 메뉴가 혼합 나열되어, 특정 카테고리만 탐색하려는 사용자의 효율이 낮았음
  • 필터 기능 부재 — 저당·저칼로리·카페인 프리 등 개인 선호 조건으로 메뉴를 좁혀볼 수 있는 필터 기능이 없어, 조건에 맞는 메뉴를 직접 확인해야 했음

💡 UX 설계 근거

  • 6개 서브 카테고리 탭 — 전체·커피·디카페인·음료·티·아이스크림으로 세분화한 서브 탭을 상단에 배치하여, 1-tap으로 원하는 카테고리로 즉시 이동 가능
  • 필터 칩 시스템 연동 — 각 서브 카테고리에서 저당·저칼로리·카페인 프리 필터 칩을 동일하게 지원하여, 카테고리 + 건강 조건의 교차 필터링 제공
  • 메뉴 카드 UI 통일 — 썸네일·메뉴명·가격·옵션을 포함한 카드 UI를 전 카테고리에 동일하게 적용하여, 일관된 탐색 경험 유지

📋 사용자 시나리오

카테고리 탐색커피/음료 탭 → '디카페인' 서브 탭 → 디카페인 메뉴만 모아보기 → 선택
조건부 탐색커피/음료 탭 → '커피' 서브 탭 → '저칼로리' 필터 칩 활성화 → 저칼로리 커피만 필터링
전체 브라우징커피/음료 탭 → '전체' 서브 탭 → 스크롤하며 전체 메뉴 훑어보기 → 관심 메뉴 발견

✅ 기대 성과
  • 메뉴 탐색 시간 단축 — 6개 서브 카테고리 분류로 불필요한 스크롤을 제거하고, 원하는 카테고리 내에서만 탐색하여 메뉴 발견 시간 대폭 단축
  • 탐색 이탈률 감소 — 필터 칩을 통한 조건부 탐색으로 선택 피로도를 낮추고, 메뉴 탐색 중 이탈 없이 주문까지 이어지는 전환율 향상 기대
  • 디카페인·건강 메뉴 발견성 향상 — 디카페인 전용 서브 탭과 건강 필터 칩을 통해 특정 니즈를 가진 사용자의 메뉴 발견 경험 개선

🍰 디저트 탭 — 디저트 전용 탐색

🎯 핵심 목적

음료와 디저트를 독립된 탭으로 분리하여, 디저트 전용 브라우징 경험을 제공하고 추가 주문을 유도하는 전용 탭입니다.

⚠️ 이전의 문제점

  • 음료·디저트 혼합 나열 — 기존 앱에서는 음료와 디저트가 같은 카테고리에 혼합되어 나열되어, 디저트만 따로 탐색하기 어려웠음
  • 디저트 발견성 저하 — 음료 위주의 메뉴 구조에서 디저트 항목은 하단에 밀려 노출이 적었고, 음료 주문 시 디저트 추가 주문을 유도하는 경로가 부재했음
  • 페어링 탐색 불편 — 음료와 함께 즐길 디저트를 찾으려면 카테고리를 전환하며 별도로 탐색해야 했으며, 음료-디저트 페어링을 자연스럽게 유도하는 UI 동선이 없었음

💡 UX 설계 근거

  • 독립 탭 분리 — 디저트를 음료와 동등한 레벨의 독립 탭으로 분리하여, 디저트 탐색의 명확성과 접근성을 대폭 향상 — 메뉴 탐색 시 자연스럽게 디저트 탭을 인지하도록 설계
  • 디저트 전용 메뉴 카드 — 음료와 동일한 카드 UI(썸네일·메뉴명·가격)를 적용하여 일관된 경험을 유지하면서, 디저트 카테고리에 집중한 탐색 동선 제공
  • 음료 탭과의 자연스러운 전환 — 커피/음료 탭에서 디저트 탭으로의 전환이 1-tap으로 가능하여, 음료 선택 후 디저트 추가 주문으로의 자연스러운 동선 유도

📋 사용자 시나리오

디저트 탐색디저트 탭 전환 → 전체 디저트 메뉴 브라우징 → 관심 디저트 선택 → 장바구니 추가
음료 + 디저트 세트커피/음료 탭에서 음료 선택 → 디저트 탭 전환 → 페어링 디저트 추가 → 함께 주문
디저트만 주문앱 진입 → 디저트 탭 직접 이동 → 디저트 선택 → 장바구니 → 결제

📈 기대 효과

✅ 기대 성과
  • 디저트 추가 주문율 향상 — 독립 탭 분리로 디저트 노출 빈도를 높이고, 음료 탭과의 자연스러운 전환 동선으로 추가 주문 유도
  • 객단가 향상 — 음료 + 디저트 세트 주문 패턴을 활성화하여 건당 평균 주문 금액 증가 기대
  • 디저트 탐색 만족도 개선 — 디저트 전용 공간에서의 집중 탐색으로 발견 효율성과 사용자 만족도 동시 향상
