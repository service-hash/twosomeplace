💡 설계 의도

🎯 이 화면의 핵심 목적

사용자가 원하는 음료를 카테고리/서브카테고리 기반으로 빠르게 찾을 수 있도록 2단계 탭 네비게이션을 설계했습니다. 메인 카테고리에서 대분류를 선택한 뒤, 서브 카테고리로 선택 범위를 점진적으로 좁혀가는 탐색 패턴입니다.

UX 설계 근거

  • 2단계 계층 구조 — 메인 카테고리(커피/음료, 디저트, 푸드, 상품) → 서브 카테고리(커피, 디카페인, 음료, 티/티라떼 등)로 선택 범위를 점진적으로 좁혀가는 탐색 패턴을 적용했습니다
  • 간결한 리스트 형태 — 각 메뉴 아이템은 썸네일 + 이름 + 가격의 간결한 리스트 형태로 스캔 용이성을 극대화하여 빠른 탐색이 가능합니다
  • 서브 카테고리 탭 — 커피, 디카페인 커피, 음료, 티/티라떼, 아이스크림/빙수 등 세분화된 탭으로 정확한 필터링을 지원합니다
  • 하단 매장 정보 바 — "어디서 수령할지"를 항상 상기시켜 주문 전 매장 확인을 유도합니다

사용자 시나리오

커피 탐색커피/음료 탭 → 커피 서브탭 → 원하는 메뉴 탭
논커피 탐색커피/음료 탭 → 음료 또는 티/티라떼 서브탭 → 선택
디카페인 탐색커피/음료 탭 → 디카페인 커피 서브탭 → 선택

🧩 화면 구성요소

상단 헤더 (Header)

좌측 뒤로가기(메뉴홈) 및 홈 버튼, 중앙 "투썸오더" 타이틀, 우측 즐겨찾기/공유/장바구니 아이콘을 배치합니다. 메뉴 홈과 동일한 고정 헤더 구조입니다.

  • 뒤로가기 버튼 클릭 시 메뉴 홈(menu.html)으로 이동
  • 장바구니 아이콘에 담긴 수량 빨간색 뱃지 표시

메인 카테고리 탭 (Main Category Tabs)

상단 헤더 아래 위치하는 1차 카테고리 수평 탭입니다. 선택된 탭 하단에 인디케이터 바가 표시되며, 현재 "커피/음료"가 활성 상태입니다.

NEW 커피/음료 디저트 푸드 상품

서브 카테고리 탭 (Sub Category Tabs)

메인 카테고리 하위에 표시되는 2차 필터 탭입니다. 메인 카테고리 선택에 따라 서브 탭 목록이 동적으로 변경됩니다. 현재 "커피"가 활성 상태입니다.

커피 디카페인 커피 음료 티/티라떼 아이스크림/빙수

메뉴 리스트 (Menu List)

선택된 서브 카테고리에 해당하는 메뉴 아이템을 리스트 형태로 표시합니다. 각 아이템은 썸네일 이미지, 메뉴명, 가격으로 구성되며 탭 시 메뉴 상세 화면으로 이동합니다.

  • 서브탭 전환 시 renderMenuList(subCategory) 호출로 리스트가 동적 갱신
  • 각 메뉴 아이템은 터치 영역이 넉넉하게 설계되어 오탭 방지
  • 썸네일 + 메뉴명 + 가격의 간결한 구성으로 스캔 용이성 극대화

플로팅 검색 버튼 (FAB)

화면 우하단 고정 빨간색 원형 버튼(52x52px). 카테고리 탐색 중에도 키워드 기반 메뉴 검색에 빠르게 접근할 수 있습니다.

하단 매장 정보 바 (Bottom Bar)

현재 선택된 주문 매장 정보와 "변경" 버튼을 표시합니다. 높이 60px, 어두운 배경으로 화면 하단에 고정되어 주문 매장을 항상 상기시킵니다.

🔗 인터랙션 & 화면 전환

진입 경로

  • 메뉴 홈의 커피/음료 카테고리 탭 클릭 시
  • 다른 카테고리 화면에서 커피/음료 탭 전환 시

이 화면에서 이동 가능한 곳

메뉴 아이템 탭→ 메뉴 상세 (menu-detail.html?id={item.id})
장바구니 아이콘→ 장바구니 (cart.html)
즐겨찾기 아이콘→ 즐겨찾기 (favorites.html)
매장 변경 버튼→ 매장 선택 (store-select.html)
뒤로가기 / 홈→ 메뉴 홈 (menu.html)

탐색 흐름

메뉴 홈 → 카테고리 탐색 → 메뉴 상세 → 장바구니

⚙️ 데이터 & 기술 스펙

메뉴 데이터는 menu-data.js의 MENU_DATA 배열을 category/subCategory 필드로 필터링하여 사용합니다.

데이터 소스menu-data.js → MENU_DATA 배열
필터링 방식category + subCategory 필드 기반 동적 필터링
리스트 렌더링서브탭 클릭 시 renderMenuList(subCategory) 호출
메뉴 이동아이템 클릭 시 menu-detail.html?id={item.id}로 이동
최대 화면 너비430px (모바일 최적화)

사용 기술

HTML5 CSS3 Vanilla JS 동적 필터링 모바일 퍼스트
